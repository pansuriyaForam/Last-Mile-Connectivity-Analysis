## 📌 Project Overview
This project presents a data-driven, mathematically rigorous pipeline to optimize feeder bus network locations along the **Hyderabad Metro Red Line (Miyapur ↔ LB Nagar)**. Using real-world General Transit Feed Specification (GTFS) data from HMRL and TGSRTC, it calculates a custom **Last-Mile Connectivity Index (LMCI)** and deploys an **Equity-First Greedy MCLP** algorithm to propose new bus stop locations, specifically targeting "Transit Deserts."

## ✨ Key Features
* **Production-Grade OOP Architecture:** Built with robust Python classes (`GTFSLoader`, `TransitOptimizer`) and type-hinting.
* **Equity-First Weighting:** Modifies standard centrality-biased algorithms by applying an inverse-LMCI weight, forcing the solver to prioritize underserved peripheral zones (e.g., Miyapur, JNTU) over already well-connected city centers.
* **Mathematical Optimality Guarantee:** Utilizes the Greedy heuristic for the Maximum Coverage Location Problem (MCLP), which is mathematically proven to achieve $\ge 63.2\%$ of the true optimal solution (Nemhauser, Wolsey & Fisher, 1978).
* **Fault-Tolerant Data Handling:** Features built-in synthetic fallback data generation. If local GTFS files are missing or paths are broken, the pipeline gracefully degrades and generates calibrated proxy data to ensure complete reproducibility.
* **High-Fidelity Visualizations:** Automatically exports 300 DPI matplotlib charts and interactive Folium heatmaps.

## 🧮 The Mathematics: LMCI & MCLP

### Last-Mile Connectivity Index (LMCI)
The LMCI evaluates the physical and temporal availability of transit at each station:
`LMCI = 10 * [0.5*(Stop Density) + 0.4*(Peak Frequency) + 0.1*(Walk-Zone Coverage)]`

### The Equity-Constrained Objective Function
Instead of maximizing raw population coverage, the objective function maximizes *weighted* coverage:
`Weight = 1 / (LMCI + 0.5)`
This ensures that adding a stop in a Transit Desert (Low LMCI) mathematically outweighs adding a redundant stop in a highly connected central hub.

## 🚀 Usage & Reproduction
1. Clone the repository.
2. Ensure you have the required libraries installed: `pip install pandas geopandas folium matplotlib scipy shapely`
3. Run the Jupyter Notebook sequentially. 
4. *Note: If you do not have the raw GTFS files in the `Data/` directories, the notebook will automatically trigger the `_make_fallback_bus()` method to allow you to review the pipeline's logic and outputs.*

## 📈 Urban Impact
By algorithmically identifying and patching transit deserts, this framework provides LTMRHL and TGSRTC planners with a millisecond-resolution tool to deploy a fixed budget of $k$ new feeder stops. Transitioning just 5–10% of short-distance auto/bike commuters to the feeder network significantly reduces urban congestion and emissions along the Red Line corridor.
